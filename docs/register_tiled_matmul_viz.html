Why 2×2 outputs per thread is faster

  

    In the compute loop, look at what the followed thread reads at each k
    (purple cells): 2 values from Ads and 2 from Bds — and it does 4 multiply-adds with them.
    Every shared-memory value it reads is used twice: Ads[ty][k] feeds both
    c00 and c01; Bds[k][tx] feeds both c00 and c10.

    Doubling FMAs per shared-memory access matters because shared memory, while fast, still has
    finite bandwidth and latency — registers are the only storage that keeps up with the ALUs.
    The accumulators c00..c11 live in registers for the whole kernel, so the inner loop's
    products land in the fastest storage available. Larger register tiles (4×4 = 16 outputs/thread)
    push the ratio higher still — until register pressure starts hurting occupancy, which is the
    trade-off that bounds this technique.
  

  

    Mapping to the real kernel: TILE_WIDTH 4 → 16, block 4×4 → 16×16 threads,
    C tile 8×8 → 32×32, inner k-loop 4 → 16 steps. Nothing else changes.
