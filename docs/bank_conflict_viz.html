Shared-Memory Bank Conflicts — and the +1 Padding Fix

Scaled-down model: 8 banks, 8 lanes per warp, tile[8][8] vs tile[8][9] (real hardware: 32 banks, 32 lanes, tile[32][32] vs tile[32][33] — identical math, since 8 % 8 = 0 just like 32 % 32 = 0, and 9 % 8 = 1 just like 33 % 32 = 1). Cell color = which bank that element lives in. Black-bordered cells = the 8 elements one warp accesses in a single instruction. The columns below each tile are the bank queues: chips stacked in the same queue must be served one per cycle — that's the conflict.

tile[ty][tx] = A[row * cols + col]; // load: warp walks a ROW of the tile __syncthreads(); B[out_row * rows + out_col] = tile[tx][ty]; // transposed read: warp walks a COLUMN

__shared__ float tile[8][8]

__shared__ float tile[8][9]  ← +1 pad

Why the +1 works

Bank of an element = (its flat index) % (number of banks). Flat index of tile[i][j] = i * rowStride + j, where rowStride is the declared row width — 8 unpadded, 9 padded.

The transposed read walks a column: lane L reads tile[L][ty], flat index L * rowStride + ty. Between consecutive lanes the address jumps by rowStride:

address stride between lanesstride % banksbanks hittransactions
tile[8][8]80all lanes → one bank8 (serialized)
tile[8][9]91each lane → next bank over1

A stride that shares a factor with the bank count keeps revisiting the same banks; a stride coprime with the bank count visits every bank before repeating. 8 (or 32) is the bank count — worst case. 9 (or 33) is coprime — best case. The pad column (hatched) is never read or written; it exists only to shift each row's starting bank by one. Cost: one float per row. Payoff: the column walk goes from 8× (32× on real hardware) serialized to a single transaction.

Switch the access selector to the load pattern tile[ty][tx] and note both tiles are conflict-free: a row walk has stride 1, which is always coprime with the bank count. Conflicts come from column walks — exactly what a transpose must do somewhere. The padding lets you keep the column walk in shared memory and make it free.
